How JavaScript Works Behind the Scenes

Execution context:

A box, a Container or a wrapper which stores variables and in which a piece of our code is evaluated and executed.

Global Execution Context:

- Code that is not inside any Function
- Associated with the global object.
- In the browser, that is the window object.

Execution context in detail:

- Variable Object(VO)
- Scope Chain

Hoisting:

Hoisting is default behaviour of JavaScript of moving declarations to the top.
In the example you can see that console log can be called before the var declartion has been made.

Scoping:

Each new function creates a scope.
Lexical Scoping: Function written inside a Function gets acces to the scope of the outer Function(Parent) including Variables

The 'This' Variable:

Regular function call: The 'This' keyword points at the global object(The window object, in the browser).
Method call: The 'This' variable points to the object that is calling the method.
The 'This' keyword is not assigned a value until a function where it is defined is actually called.
